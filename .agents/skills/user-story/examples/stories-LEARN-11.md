## STORY: Upload course materials  (LEARN-21)
**Epic:** LEARN-11 · **Release:** MVP · **Estimate:** 3 · **Labels:** FA-2, MVP

### Story
As a **teacher**, I want **to upload my course materials in common formats**, so that **the
tutor can ground its guidance in my actual curriculum**.

### Acceptance Criteria
**Scenario: Accepted upload**
- Given a signed-in teacher on a subject page
- When they upload a PDF, DOCX, or TXT file up to 25 MB
- Then the file is accepted and its status shows "processing" within 2 seconds

**Scenario: Unsupported format rejected**
- Given a signed-in teacher
- When they upload a file that is not PDF, DOCX, or TXT
- Then the upload is rejected with the message "Unsupported file type" and nothing is stored

**Scenario: Oversized file rejected**
- Given a signed-in teacher
- When they upload a file larger than 25 MB
- Then it is rejected with the message "File exceeds 25 MB limit" and nothing is stored

### Covered requirements
- FR-004

### Out of scope
Editing or versioning already-uploaded materials; bulk folder upload.

## STORY: Prepare materials for retrieval  (LEARN-22)
**Epic:** LEARN-11 · **Release:** MVP · **Estimate:** 5 · **Labels:** FA-2, MVP

### Story
As a **teacher**, I want **my uploaded materials to be processed into a searchable form**, so
that **the tutor can find the right passage to ground each answer**.

### Acceptance Criteria
**Scenario: Material becomes retrievable**
- Given a teacher has uploaded a valid document
- When processing completes
- Then the document's content is stored as retrievable chunks and its status shows "ready"

**Scenario: Structural content preserved**
- Given a document containing an equation or table
- When it is processed
- Then that equation or table appears intact within at least one retrievable chunk

**Scenario: Processing failure surfaced**
- Given a document that cannot be processed (e.g. corrupt file)
- When processing fails
- Then the status shows "failed" with a reason, and the teacher can retry or remove it

### Covered requirements
- FR-005

### Out of scope
The embedding/index configuration itself (handled by the retrieval story and infra); tuning
chunk size.

## STORY: Ground tutoring responses in retrieved material  (LEARN-23)
**Epic:** LEARN-11 · **Release:** MVP · **Estimate:** 5 · **Labels:** FA-2, MVP

### Story
As a **student**, I want **the tutor's guidance to be based on my teacher's materials**, so
that **the help I get matches what I'm actually being taught**.

### Acceptance Criteria
**Scenario: Relevant context retrieved**
- Given a subject with processed materials
- When a student asks a question about a covered topic
- Then the tutor retrieves the most relevant material before responding

**Scenario: Response stays grounded**
- Given relevant material has been retrieved
- When the tutor responds
- Then its guidance is consistent with the retrieved material and does not introduce outside
  facts not present in the curriculum

**Scenario: No relevant material**
- Given a student asks about a topic not covered by any uploaded material
- When the tutor responds
- Then it indicates the topic isn't in the course materials rather than inventing content

### Covered requirements
- FR-006

### Out of scope
The verifier/regeneration mechanism (epic LEARN-13); diagram generation (LEARN-12).
